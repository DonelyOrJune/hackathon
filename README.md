为了学习到新的编程知识，我带着纯粹的学习想法，不在乎是否能赢得比赛奖励，抱着试一试的心情来参加编程马拉松比赛。我是计算机专业的学生，本身具有一定的编程基础，但并不意味着我学习新东西起来就很容易，只能说过程相比其他没有基础的人可能要顺利一点吧，在理解上也比较容易弄懂。编程无非既是运用自己编写的代码来完成电脑上各种自己希望做出来的操作，我并没有特别的看法与改观，学习过程也完全是按部就班，唯有一些小地方会得到意外的惊喜（比如从网页的审查元素上看到度娘给的可爱的小彩蛋23333）。视频教学在我看来是很常规的操作，我觉得本次编程马拉松最大的特点就是，跟我本专业的专业课上教授的知识有得一比，很基础，但是感觉知识点太零碎，如果可以专一一下学html或者css的所有知识就好了（不过也不可能啦毕竟比赛时间有限hhhh）。

在比赛中，我不仅获得了新的编程知识，还增强了与人沟通交流的能力（比如跟林熙老师聊得还不错呀，还跟师兄讨论了相关专业知识）。在本次比赛的一个月内，其实我是非常忙的，作为新一届国际学院学生会的一份子，不仅要顾及自己的学业，顾及部门的工作，还要顾及对师弟师妹们工作能力的传授与指导，实际上我并没有花太多时间在编程马拉松上面，平时因为晚课是没有去到线下培训现场的――这也是我感到很遗憾的一件事。每个周末抽个几小时来完成每个站点，有不懂的就问同学跟老师，没想过要放弃啥的，也不需要啥支撑。。。学海无涯苦作舟，学习，这个长跑之路本身都是需要坚持的。

我不太了解工作人员那边的事呢，就觉得他们好辛苦，策划筹备，还要应对同学们问的问题，很不容易阿！谢谢所有的工作人员，给予了我们这样一个学习编程的机会！

如果还有下一个月的比xue拼xi，我当然会参加啦！学习是一个连续的过程呀，只学了一个月肯定是不够的，要永远地学下去。即便没有下个月的比拼，我还是希望能有像比赛这样的压力去推动我学习更多的东西啦！因为自己学习的话学习可能就不够专业，学到的东西也没那么系统，还是得跟着老师或者教程一步步来学呀。

我想对自己说的话，就是，如果能把参赛的精神放到自己的专业学习上就好了。因为我感觉专业课程比编程马拉松还要难呀，什么java、计算机组成原理，这两门课不能只靠老师，要自己课后多花时间研究，可我自己，一到了周末，只能自己望着书发愁hhhhhhh好难好难，必须要更加多花时间多花精力了呀！
